第一章 走进JAVASCRIPT黑洞 ​

开启控制台：command+option+j

变量声明基础知识:声明了一个叫做hdcms的web变量，可以使用console.log查看

html
var web ="houdunren";
console.log(web);

声明变量不需要像其他语言声明变量类型，它根据你给出的值自动变化:

html
 var web= {},name = "houdunren",url=100;
        console.log(typeof url);
        console.log(typeof name);
        console.log(typeof web);

解析过程与变量提升：使用var声明，把变量声明放在查询后面，会出现变量查询为空，因为解析器先分析一遍代码，变量提升，然后按顺序输出。使用let或者const声明，会出现报错。所以后者才是我们需要的。

var、let、const共同点：函数中可以访问外部全局变量，在自己的区域内别人外部不可以访问。

全局污染：没有使用声明变量就会出现污染，导致问题出现，可以使用严格模式"use strict";严格声明

块作用域：var没有块作用域，会影响全局，而let有块作用域，不会影响全局

html
<script>
        {
            let web ="houdunren";

        }
        console.log(web);
    </script>

此时访问不到，因为let有块作用域，换成var就可以了，相当于全局定义，放在函数中也一样。推荐使用块作用域，防止了全局影响。

const常量声明：推荐使用大写，常量声明不可以在同一个作用域更改：

html
<script>
        const URL = "http://houdunren.com/apo/v1";
 				URL ="hdcms";
    </script>

此时会报错，在同一个块作用域不可以修改常量变量。

而下面在函数中有定义URl，此时他们在不同的块作用域中所以可以再次声明。

html
 <script>
        const URL = "http://houdunren.com/apo/v1";
        function show(){
            const URL="http://houdunren.com/api/v1"
            console.log(URL);
        }
        show();
    </script>

冻结变量：锁住一个变量让他无法修改如下例

html
 <script>
        "use strict"
        const HOST ={
            url:"http://houdunren.com/api",
            port: 443
        };
        Object.freeze(HOST);
        // HOST.port =80;
        console.log(HOST);
    </script>

同时使用了严格声明，对下面的修改产生报错提示，不然的话使用冻结变量，不会提示你下面修改失效，让你没有察觉。

传值与传址：传送的是一个简单的值，还是地址

html
<script>
        let a=1;
        let b=a;
        console.log(a,b);
        b=3;
        console.log(a,b);
        let e={name:"houdunren"};
        let f=e;
        console.log(e,f);
        f.name="hdcms";
        console.log(e,f);

    </script>

严格模式高质量代码卫士：使用"use strict"影响块区域内，作用域是向下查找。
